Порушені принципи SOLID:
    Принцип єдиного обов'язку - клас MSDataBase відповідає за доступ до даних, логіку операцій і конкретну реалізацію БД;
    Принцип відкритості/закритості - система не відкрита для розширення;
    Принцип інверсії залежностей - код залежить від реалізацій, а не абстракцій.
    Принцип розділення інтерфейсу - клієнт залежить від методів, які не використовує.

Архітектурні проблеми:
    Жорстка прив’язку до конкретної БД.
    Відсутність абстракції.
    Погана масштабованість.
